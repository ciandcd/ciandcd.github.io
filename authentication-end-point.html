Authentication end-point

From:http://www.go.cd/2015/06/18/authentication-end-point.html

Starting 15.2.0 Go Server will expose authentication end-point. What this means is Go users can add "custom" authentication schemes through plugins. With plugin settings & web request handling ability plugin developers get enough flexibility to write any authentication plugin they intend to write.

Examples of integrations possible:

  • OAuth Login - GitHub, Google, Hotmail, Yahoo! etc.
  • Single Sign-on (SSO) - LDAP, Okta etc.
  • 2-factor authentication - SMS verification etc.
  • Custom username & password authentication
